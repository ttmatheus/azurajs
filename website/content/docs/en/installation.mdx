## TypeScript Configuration ğŸ”§

AzuraJS uses TypeScript decorators, so we need to configure the compiler correctly.

### Recommended Configuration

Create or update your `tsconfig.json` with these settings:

```json title="tsconfig.json"
{
  "compilerOptions": {
    // Required settings for decorators
    "experimentalDecorators": true,      // âš ï¸ REQUIRED
    "emitDecoratorMetadata": true,       // âš ï¸ REQUIRED
    
    // Recommended settings
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",       // or "node16"
    
    // Code quality
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    
    // Output
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

### Understanding the Settings

<Callout type="info" title="experimentalDecorators">
**What it does:** Enables TypeScript decorator support.

**Why it's necessary:** Without this, `@Controller`, `@Get`, `@Post` and other decorators won't work.
</Callout>

<Callout type="info" title="emitDecoratorMetadata">
**What it does:** Allows TypeScript to emit metadata about decorators.

**Why it's recommended:** Required for advanced features like dependency injection and automatic type validation.
</Callout>

<Callout type="tip">
**Tip:** If you're using Node.js, you can use `"moduleResolution": "node16"` instead of `"bundler"`.
</Callout>

## Troubleshooting ğŸ”

Found a problem? Here are solutions to the most common issues:

### âŒ Error: "Decorator is not recognized"

**Problem:** Decorators like `@Controller` cause syntax error.

**Solution:**

1. Check `tsconfig.json`:
   ```json
   {
     "compilerOptions": {
       "experimentalDecorators": true,
       "emitDecoratorMetadata": true
     }
   }
   ```

2. Verify TypeScript version:
   ```bash
   npx tsc --version  # should be 5.0.0+
   ```

3. Restart your editor (VS Code, etc.)

### âŒ Error: "Cannot find module 'azurajs'"

**Problem:** TypeScript can't find the AzuraJS module.

**Solution:**

1. Check if it's installed:
   ```bash
   npm list azurajs
   ```

2. Reinstall if needed:
   ```bash
   rm -rf node_modules package-lock.json
   npm install
   ```

3. Check `moduleResolution` in tsconfig:
   ```json
   {
     "compilerOptions": {
       "moduleResolution": "bundler"  // or "node16"
     }
   }
   ```

### âŒ Type Errors

**Problem:** TypeScript complains about incompatible types.

**Solution:**

1. Install type definitions:
   ```bash
   npm install -D @types/node
   ```

2. Configure tsconfig correctly:
   ```json
   {
     "compilerOptions": {
       "esModuleInterop": true,
       "skipLibCheck": true
     }
   }
   ```

3. Clear TypeScript cache:
   ```bash
   rm -rf .tsbuildinfo dist
   npx tsc --build --clean
   ```

### âŒ Port already in use

**Problem:** Error `EADDRINUSE: address already in use`.

**Solution:**

<Tabs items={['Windows', 'Linux/Mac']}>
  <Tab value="Windows">
    ```bash
    # Find the process using the port
    netstat -ano | findstr :3000
    
    # Kill the process (replace PID)
    taskkill /PID <PID> /F
    ```
  </Tab>
  
  <Tab value="Linux/Mac">
    ```bash
    # Find and kill the process
    lsof -ti:3000 | xargs kill -9
    ```
  </Tab>
</Tabs>

### âŒ Import path doesn't work

**Problem:** `import { X } from "azurajs/decorators"` doesn't work.

**Solution:**

Make sure you're using the correct path:

```typescript
// âœ… Correct
import { Controller, Get } from "azurajs/decorators";
import { AzuraClient } from "azurajs";

// âŒ Incorrect  
import { Controller } from "azurajs";  // won't work
```
